# ALFRED 키오스크 UI — 핵심 코드 설명 (수퍼바이저 리뷰용)

> 각 파일 상단에도 동일한 내용의 `[한글 해설]` 주석이 달려 있어, 코드를 열면
> 바로 맥락을 읽을 수 있습니다. 아키텍처는 **"화면은 상태머신이, 데이터는
> 서비스가"** 한 문장으로 요약됩니다.

## 아키텍처 한눈에

```
.env (VITE_FLOOR, VITE_USE_MOCKS, VITE_ROSBRIDGE_HOST …)
  └─ config/env.ts ──▶ config/kiosk.config.ts (층 → robot2/robot4 프로필)
       └─ services/createServices.ts  ← 목/실물 구현을 고르는 유일한 곳
            ├─ STT(Soniox) · LLM(Claude) · TTS(브라우저)
            └─ rosbridge 서버(허브 노트북, ws://호스트:9090) 위에 3채널 — 별도 FMS 관제 서버 없음
                 ├─ IF-01 발행  /information            → alfred_driving/web_request_node 가 수신·실행
                 ├─ 감지 구독   /robotN/detection/info   ← 비전 트랙 (alfred_vision/detector_node)
                 └─ 상태 구독   /robotN/ui_state         ← alfred_bridge/escort_state_bridge_node 가 발행
app/App.tsx (Provider 조립) ─▶ KioskApp(셸) ─▶ KioskRouter ─▶ 화면 8종
core/kiosk/kioskMachine.ts  ← 모든 화면 전환이 통과하는 순수 리듀서
```

## 1순위 — 발표 때 꼭 보여줄 파일 4개

### `src/core/kiosk/kioskMachine.ts` — 상태머신 (이 앱의 심장)
8개 화면(patrol/home/map/voice/guiding/alert/charging/waiting)을 유한 상태로 두고,
18종 이벤트를 **순수 함수 하나**로 처리한다. 잘못된 전이는 no-op이라 늦게 도착한
이벤트(취소 직후의 진행률 등)에도 UI가 깨지지 않는다. 안전 가드가 핵심:
- 유휴 복귀가 **안내중을 절대 끊지 않음** (`IDLE_TIMEOUT`)
- **경보 중에는 로봇 화면 전환 차단** (`ENTER_CHARGING/WAITING`, `ROBOT_ESCORT`)
- 로봇 PATROL 보고가 **로컬 안내·도착 화면("도착했어요!")을 끊지 않고**, 동시에
  **남아있던 비상경보는 해제** (`EXIT_ROBOT_SCREEN` — `alert` 포함)

### `src/features/guiding/GuidanceProvider.tsx` — 에스코트 오케스트레이터
안내의 시작(`guideTo`)/취소(`cancelGuidance`)를 한 곳에서 관리. 시설이 확정되면
IF-01 ESCORT를 발행하고, **실로봇 연결 시에는 "준비 중" 화면만 띄운 뒤 로봇의
IF-02 상태가 화면을 주도**하게 한다(화면이 스스로 완료되지 않음). 오프라인/목
모드에서는 로컬 시뮬레이션이 진행률→도착→순찰 복귀까지 대신한다. VI 모드면
고객 프로필을 blind로 실어 보낸다.

### `src/services/createServices.ts` — 목/실물 스위치 (합성 루트)
`VITE_USE_MOCKS=true`면 STT·LLM·IF-01 송신·감지·로봇상태 전부 목으로 → **마이크·
API 키·로봇·서버 없이 완전 오프라인 데모 가능**. 목 해제 + rosbridge 호스트 설정 시
WebSocket 1개 위에 3채널이 실물로 전환된다. "어떤 구현을 쓸지"는 이 파일 밖에서
절대 결정되지 않는다 — 화면 코드는 목/실물을 모른다.
(코드의 `FmsService`는 IF-01 송신 채널의 관례적 이름 — 실제 수신자는 rosbridge
너머의 `web_request_node`이며 별도 FMS 서버는 없다.)

### `src/core/kiosk/robotStatus.ts` — IF-02 → 화면 매핑표
로봇이 보내는 상태 문자열 13종을 화면 이벤트로 번역하는 유일한 지점.
주행 트랙과의 협업 시 "로봇 상태가 화면에 어떻게 반영되나?"는 이 파일 하나로
설명이 끝난다. (ESCORT_1F/2F는 poi_id→시설명 해석, ESCORT_COMPLETED는 "도착했어요!"
표시·타이머 때문에 `RobotStateProvider`에서 직접 처리)

## 데모 치트시트 (로봇 없이 전 화면 시연)

브라우저 콘솔(F12)에서:
```js
window.alfredRobotStatus('DOCKING')                  // 충전 화면
window.alfredRobotStatus('WAITING_1F')               // 대기 화면
window.alfredRobotStatus({ state:'ESCORT_1F', destination:{ poi_id:'WC' }, progress:0.5 })  // 로봇 주도 안내
window.alfredRobotStatus('ESCORT_1F_FINISHED')       // "2층으로 이동하세요" 환승
window.alfredRobotStatus('ESCORT_COMPLETED')         // "도착했어요!" 후 순찰 복귀
window.alfredAlert('FIRE')                           // 화재 경보 (INJURED/SUSPICIOUS 동일)
window.alfredClearAlert()                            // 경보 해제
```
목(mock) 모드에서는 음성안내의 STT도 데모 문장("화장실 어디예요" 등)을 자동
재생하므로 마이크 없이 전체 음성 플로우를 보여줄 수 있다.
