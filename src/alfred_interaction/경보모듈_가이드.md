# 비상 경보 모듈 가이드 (YOLO 탐지 → 전체화면 경보)

> requirement ver03 — patrol 중 로봇이 **화재 / 부상자 / 거동수상자**를 YOLO로 감지하면,
> 그 토픽을 ros_bridge로 받아 키오스크에 **전체화면 경보 + 사이렌 + TTS**를 띄웁니다.
> UI 쪽은 완성돼 있고, **담당자는 아래 두 곳만 연결**하면 됩니다.

## 1. 담당자가 연결할 곳 (딱 2개)

### ① 어떤 토픽으로 받을지 — `.env`
```bash
VITE_DETECTION_TOPIC=/detection          # 쉼표로 여러 개 가능: /yolo/fire,/yolo/intruder
```
키오스크는 이 토픽(들)을 ros_bridge로 **구독**합니다. (rosbridge 호스트는 `VITE_ROSBRIDGE_HOST`)

### ② 메시지 → 종류 매핑 — `src/services/detection/RosBridgeDetectionService.ts`
`extractDetection()` / `labelFromMessage()` 가 들어온 메시지에서 종류를 뽑습니다. 지금 지원하는 형태(아무거나 OK):
| 보내는 형태 | 예시 |
|---|---|
| `std_msgs/String` (라벨) | `{ data: "FIRE" }` |
| `std_msgs/String` (JSON) | `{ data: "{\"type\":\"FIRE\"}" }` |
| 커스텀 메시지 필드 | `{ type / event_type / label / class: "FIRE" }` |
| 토픽 이름으로 구분 | `/detection/fire` 처럼 토픽명에 키워드 |

라벨 키워드 매핑은 `src/core/domain/detection.ts`의 `parseDetectionLabel()` 한 곳에서 바꿉니다
(`fire/화재`, `injured/부상자`, `suspicious/거동수상자` 등 — 실제 라벨 확정되면 여기만 수정).

## 2. 종류별 동작 (커스터마이징: `src/features/alerts/alerts.config.ts`)
| 종류 | 이모지 | 사이렌 | TTS(큰 소리, 반복) |
|---|---|---|---|
| `FIRE` 화재 | 🔥 | 119(fire) | "화재발생 화재발생 즉시 안전한 곳으로 대피하십시오" |
| `INJURED` 부상자 | 🚒 | 없음 | "역사 내 부상자 발생 119에 신고해주세요" |
| `SUSPICIOUS` 거동수상자 | 🚨 | 112(police) | "거동수상자감지 거동수상자감지 112에 신고하겠습니다 잡았다 요놈 잡았다 요놈" |

- 화면: 종류별 색으로 **번쩍이는 전체화면** + 큰 이모지/문구, 하단 **"경보 해제"** 버튼(직원이 누르면 patrol 복귀).
- 소리: **사이렌 burst → TTS → 잠깐 쉼 → 반복** (겹치지 않아 음성이 또렷). 사이렌은 Web Audio로 합성(음원 파일 불필요).

## 3. 테스트

### 로봇 없이 (UI만) — 브라우저 콘솔(F12)
```js
window.alfredAlert('FIRE')        // 또는 'INJURED' / 'SUSPICIOUS'
window.alfredClearAlert()         // 해제
```

### 실제 ros_bridge 경로 — rosbridge 호스트(노트북)에서
```bash
ros2 topic pub /detection std_msgs/msg/String "{data: 'FIRE'}" -r 1
```
→ 키오스크에 화재 경보가 떠야 정상. (호스트 rosbridge가 그 토픽을 **그래프에서 봐야** 함 —
`ros2 topic list`에 `/detection`이 보이면 OK. 안 보이면 rosbridge를 `ROS_SUPER_CLIENT=True`로 재기동.)

## 4. 모듈 구조 (어디를 보면 되나)
```
src/core/domain/detection.ts            # DetectionType + 라벨 파서 (도메인)
src/services/detection/                 # 수신 채널 (rosbridge subscribe) ← ② 연결 지점
  ├─ DetectionService.ts                #   인터페이스
  ├─ RosBridgeDetectionService.ts       #   실제(공유 RosBridgeClient)
  └─ MockDetectionService.ts            #   오프라인(no-op, 수동 트리거만)
src/features/alerts/                    # 표현 (화면/소리/문구)
  ├─ alerts.config.ts                   #   종류별 이모지/사이렌/문구 ← 커스터마이징
  ├─ AlertScreen.tsx + .module.css      #   전체화면 경보 + 오디오 루프
  └─ AlertsProvider.tsx                 #   탐지 구독 → 상태머신 dispatch + 콘솔 트리거
src/core/audio/sfx.ts                   # playSiren('fire'|'police')
src/core/kiosk/                         # 'alert' 화면 + DETECTION/CLEAR_ALERT 이벤트
```
연결은 전부 `src/services/createServices.ts`(합성 루트) 한 곳에서 이뤄집니다. FMS 송신과 **같은 rosbridge 소켓**을 공유합니다.
